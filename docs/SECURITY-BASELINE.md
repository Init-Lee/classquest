# Security Baseline for V1.5

V1.5 是学校试点系统，不是公开 SaaS。目标是最低可安全运行。

## 必须具备

1. 生产环境只使用 HTTPS。
2. 教师/API 操作必须认证。
3. 服务器不存学生真实身份，使用 `learner_hash` / `participant_hash`。
4. 学生编辑令牌只返回一次，服务端只存 hash。
5. 文件上传必须限制大小、MIME、扩展名，并使用随机存储名和 SHA-256。
6. 上传目录不能作为公开静态目录浏览。
7. CORS 只允许批准的前端域名。
8. SQLite、uploads、exports 必须备份。

## 暂缓能力

Moodle SSO、细粒度 RBAC、完整审计后台、密钥轮换系统、字段级加密。

## 日志要求

记录教师登录、审核操作、试答轮次开关、统计重算、上传失败。

不要记录明文密码、明文编辑令牌和包含敏感内容的完整请求体。
