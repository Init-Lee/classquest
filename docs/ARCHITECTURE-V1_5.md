# ClassQuest V1.5 架构

V1.5 的目标是在保留模块 3 本地优先模式的基础上，为模块 4 提供第一个可部署的轻量运行时架构。

```text
学生 / 画廊前端
  ↓
OSS 静态托管
  ↓ HTTPS JSON API
轻量服务器
  ├── Nginx
  ├── FastAPI
  ├── SQLite
  ├── 本地运行时文件
  └── 备份脚本
```

## 前端边界

- `src/platform/`：门户、全局路由、模块注册表。
- `src/modules/module-3-ai-science-station/`：已完成的模块 3。
- `src/modules/module-4-ai-info-detective/`：模块 4，占位后逐课开发。
- `src/shared/`：业务无关 UI 与纯工具。

## 后端边界

后端是模块 4 的轻量运行时层，不是 LMS，也不替代 Moodle。

负责：提交、审核、试答、评分、统计、画廊导出。

不负责：完整学生账号、Moodle SSO、WebSocket、实时协作、长期分析平台。

## 存储策略

- 学习进度仍以浏览器本地和导出的学习包为主。
- 模块 4 运行时结构化数据进入 SQLite。
- 上传与导出文件先放轻量服务器本地磁盘。
- 后续如需迁移 OSS，应通过存储适配层替换，不穿透业务层。
